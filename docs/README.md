# Documentation

Reference material for working in this repo.

## Contents

- **[Architecture](./architecture.md)** — How the pnpm workspace monorepo
  is organized and why. Read this first if you've never worked in a
  monorepo before.
- **[specs/](./specs/)** — Design specifications, one per pattern.
  Captures *what* we're building and *why*.
- **[plans/](./plans/)** — Implementation plans, one per pattern.
  Captures *how* we build it, step by step.

## Reading order for a new pattern

1. Open the spec in `specs/` — understand the goal and scope.
2. Open the plan in `plans/` — follow the task list to implement it.
3. Cross-reference with `architecture.md` if you hit anything
   monorepo-shaped.
